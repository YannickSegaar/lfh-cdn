Part 1 — Architecture Overview

How It Works

WordPress loads one script tag. That script bootstraps the entire AI agent: context detection, VoiceFlow widget, 20 custom extensions, styling, and cross-page awareness.

Key insight: WordPress only needs one line of code. Everything else lives on YOUR GitHub repo. When you update something, push to GitHub — no WordPress access needed.

Part 4 — Pre-Flight Verification

Before the Meeting with Gwyn

1 Verify the CDN is live

Open each URL in your browser. Each should load (not 404):

  • https://yannicksegaar.github.io/lfh-cdn/loader.js
  • https://yannicksegaar.github.io/lfh-cdn/css/LFH_styles.css
  • https://yannicksegaar.github.io/lfh-cdn/extensions/lfh-lead-form.js
  • https://yannicksegaar.github.io/lfh-cdn/images/LFH_Logo_FullName_White.svg
  • https://yannicksegaar.github.io/lfh-cdn/fonts/NexaRustSansBlack2.woff2

2 Test on a local page

# Create a quick test page
cat > /tmp/lfh-cdn-test.html << 'EOF'
<!DOCTYPE html>
<html><head><title>LFH CDN Test</title></head>
<body>
<h1>LFH CDN Test Page</h1>
<script type="module"
  src="https://yannicksegaar.github.io/lfh-cdn/loader.js?v=1.1.0"
  crossorigin></script>
</body></html>
EOF

# Serve it (ES modules need a server, not file://)
cd /tmp && python3 -m http.server 8080

# Open: http://localhost:8080/lfh-cdn-test.html

3 Verify each extension works

  • Chat widget bubble appears in bottom-right (give it 5-10 seconds)
  • Proactive message: "Curious about heliskiing in Northern BC?"
  • Welcome grid with quick-start options appears on open
  • "Show me your tours" → tour grid appears
  • "Compare the lodges" → lodge cards appear
  • "What are the snow conditions?" → weather widget appears
  • "I want to book a trip" → lead form eventually appears
  • Thumbs up/down appears after agent responses

4 Check browser console

Green Flags
[LFH] Extensions loaded: 9 — all extensions loaded successfully. No red errors from lfh-cdn.
Red Flags
Failed to load module script, CORS error, or net::ERR_ABORTED 404 — a file is missing or misconfigured.

5 Test fallback mode

Change URL to loader.js?v=1.1.0&fallback=true. Only lead form + feedback should work. Tour explorer, lodges, weather, hub — these should NOT appear.

6 Test on mobile

  • Widget bubble appears and is tappable
  • Chat opens full-screen on mobile
  • Extensions render correctly on small screens
  • Can scroll within modals

Part 6 — Branch Strategy & Safe Updates

Never Push Directly to Main

The main branch is production — what the live website loads. It's branch-protected: direct pushes are blocked. All changes go through staging first.

Step-by-Step Update Workflow

# 1. Switch to staging branch
cd ~/my_projects/lfh-cdn
git checkout staging
git pull origin staging

# 2. Make your changes
#    (edit extension files, loader.js, CSS, etc.)

# 3. Test locally
python3 -m http.server 8080
#    Open http://localhost:8080 and verify

# 4. Commit and push to staging
git add extensions/lfh-tours-grid.js  # (specific files, not -A)
git commit -m "Description of what changed"
git push origin staging

# 5. Test on staging URL (if configured)

# 6. Create PR to main
gh pr create --base main --head staging \
  --title "v1.x.0: Description"

# 7. Merge PR on GitHub
gh pr merge --merge

# 8. Tag the release
git checkout main && git pull
git tag v1.x.0 && git push --tags

# 9. Update CHANGELOG.md
Why this matters: Before v1.1.0, pushing to main went live instantly. One typo = broken widget for all visitors. The staging branch gives you a safety net to test before going live.

Using the URL paths Gwyn gave you, update detectPageType() in loader.js (around line 63). Example:

// If Gwyn says tours is at /heliskiing-tours/
if (p.includes('/tour') || p.includes('/heli-ski') || p.includes('/heliskiing'))
  return 'tours';

VoiceFlow Cross-Page Awareness (optional, can do later)

Already built into loader.js. To enable in VoiceFlow:

  1. Go to the main Listen/Conversation node in VoiceFlow
  2. Add an Event Trigger: page_context_update
  3. Connect to a Code Step with this code:
var data = (last_event && last_event.payload)
  ? last_event.payload : last_event;
if (data) {
  if (data.page_type) page_type = data.page_type;
  if (data.page_path) page_path = data.page_path;
}

Response Options (fastest first)

Fallback Mode (30 sec)
Tell Gwyn: "Add &fallback=true to the script URL." Result: simple text chat + lead form only.
Roll Back Code (2 min)
git revert HEAD && git push origin main — undoes the last commit. Wait 2-3 min for GitHub Pages.
Remove Entirely (10 sec)
Tell Gwyn: "Delete the script tag from the footer." Chat disappears. No side effects on the website.

Naming Convention

Format: lfh-{feature}.js — no version numbers, no "unified" suffixes.

Version tracked in git tags (v1.1.0) and file header comments. Each file has a header documenting its origin, trace types, and dependencies.
